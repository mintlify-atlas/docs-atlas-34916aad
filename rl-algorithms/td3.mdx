Twin Delayed Deep Deterministic Policy Gradient (TD3) extends [DDPG](/rl-algorithms/ddpg) with three techniques that substantially improve stability and performance: **Clipped Double Q-Learning** (two critic networks, take the minimum) to reduce overestimation, **Delayed Policy Updates** (actor updated less frequently than critics) to reduce variance, and **Target Policy Smoothing** (noise added to target actions) to prevent the policy from exploiting sharp Q-value peaks.

**Original paper:**

- [Addressing Function Approximation Error in Actor-Critic Methods](https://arxiv.org/abs/1802.09477) — Fujimoto et al., 2018

## `td3_continuous_action.py`

This variant targets continuous control environments with `Box` observation and action spaces. It is based on [sfujim/TD3](https://github.com/sfujim/TD3), with support for asymmetric action space bounds.

### Key implementation details

CleanRL's TD3 uses two separate Q-network objects (unlike the original which wraps both in one `Critic` class) and handles asymmetric action bounds by computing `action_scale` and `action_bias`:

```python
class Actor(nn.Module):
    def __init__(self, env):
        ...
        # action rescaling
        self.register_buffer("action_scale", torch.FloatTensor((env.action_space.high - env.action_space.low) / 2.0))
        self.register_buffer("action_bias", torch.FloatTensor((env.action_space.high + env.action_space.low) / 2.0))

    def forward(self, x):
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        x = torch.tanh(self.fc_mu(x))
        return x * self.action_scale + self.action_bias
```

Exploration noise is also centered at `action_bias` and scaled by `action_scale * exploration_noise`, so it works correctly for environments with non-standard action bounds (e.g., `Humanoid-v2` with `[-0.4, 0.4]`).

<Note>
  The original TD3 paper used MuJoCo v1 environments and evaluated with deterministic policy (no exploration noise). CleanRL reports training return with MuJoCo v4 environments.
</Note>

## `td3_continuous_action_jax.py`

This variant uses [JAX](https://github.com/google/jax), [Flax](https://github.com/google/flax), and [Optax](https://github.com/deepmind/optax) instead of PyTorch. It achieves roughly 2.5–4× faster wall-clock training than the PyTorch variant.

### Installation and usage

<Tabs>
  <Tab title="uv">
    ```bash
    uv pip install ".[mujoco,jax]"
    uv run python cleanrl/td3_continuous_action_jax.py --help
    uv run python cleanrl/td3_continuous_action_jax.py --env-id Hopper-v4
    uv run python cleanrl/td3_continuous_action_jax.py --env-id HalfCheetah-v4
    ```
  </Tab>
  <Tab title="pip">
    ```bash
    pip install -r requirements/requirements-mujoco.txt
    pip install -r requirements/requirements-jax.txt
    python cleanrl/td3_continuous_action_jax.py --help
    python cleanrl/td3_continuous_action_jax.py --env-id Hopper-v4
    ```
  </Tab>
</Tabs>

<Warning>
  JAX does not support Windows natively. The JAX team recommends using Windows Subsystem for Linux (WSL) to run JAX-based variants.
</Warning>

The hyperparameters and logged metrics are identical to `td3_continuous_action.py`. See the sections above for details.
