Soft Actor-Critic (SAC) is an off-policy actor-critic algorithm that maximizes a trade-off between expected return and policy entropy. Unlike [DDPG](/rl-algorithms/ddpg) and [TD3](/rl-algorithms/td3), SAC uses a stochastic policy, which enables natural exploration and can represent multi-modal optimal behaviors. SAC also uses two soft Q-networks to reduce overestimation bias, and supports automatic tuning of the entropy coefficient α.

**Original papers:**

- [Reinforcement Learning with Deep Energy-Based Policies](https://arxiv.org/abs/1702.08165)
- [Soft Actor-Critic: Off-Policy Maximum Entropy Deep Reinforcement Learning with a Stochastic Actor](https://arxiv.org/abs/1801.01290)
- [Soft Actor-Critic Algorithms and Applications](https://arxiv.org/abs/1812.05905)
- [Soft Actor-Critic for Discrete Action Settings](https://arxiv.org/abs/1910.07207)

## `sac_continuous_action.py`

This variant targets continuous control environments with `Box` observation and action spaces. It uses a numerically stable stochastic actor based on the [SpinningUp](https://github.com/openai/spinningup) implementation.

### Key implementation details

The actor uses a numerically stable log-std clamping scheme (note `LOG_STD_MIN = -5`, not `-20` as in SpinningUp):

```python
LOG_STD_MAX = 2
LOG_STD_MIN = -5

def get_action(self, x):
    mean, log_std = self(x)
    std = log_std.exp()
    normal = torch.distributions.Normal(mean, std)
    x_t = normal.rsample()  # reparameterization trick
    y_t = torch.tanh(x_t)
    action = y_t * self.action_scale + self.action_bias
    log_prob = normal.log_prob(x_t)
    # Enforcing Action Bound
    log_prob -= torch.log(self.action_scale * (1 - y_t.pow(2)) + 1e-6)
    log_prob = log_prob.sum(1, keepdim=True)
    return action, log_prob, mean
```

When `--autotune` is enabled, the entropy coefficient α is optimized using the Q-network learning rate:

```python
if args.autotune:
    target_entropy = -torch.prod(torch.Tensor(envs.single_action_space.shape).to(device)).item()
    log_alpha = torch.zeros(1, requires_grad=True, device=device)
    alpha = log_alpha.exp().item()
    a_optimizer = optim.Adam([log_alpha], lr=args.q_lr)
else:
    alpha = args.alpha
```

## `sac_atari.py`

This variant adapts SAC to discrete action spaces using a categorical policy and shared Nature-CNN network. It uses action-probability-weighted objectives throughout (critic, actor, and α tuning) to reduce gradient variance.

### Installation and usage

    ```bash
    uv run python cleanrl/sac_atari.py --env-id BreakoutNoFrameskip-v4
    # Disable automatic entropy tuning
    uv run python cleanrl/sac_atari.py --env-id PongNoFrameskip-v4 --autotune False --alpha 0.2
    ```
  </Tab>

### Key implementation details

In the discrete setting the Q-function maps states to Q-values for all actions, so full action distributions are used instead of Monte Carlo sampling. This reduces gradient variance in both the critic and actor:

```python
# Critic target using full action distribution
min_qf_next_target = next_state_action_probs * (
    torch.min(qf1_next_target, qf2_next_target) - alpha * next_state_log_pi
)

# Actor loss using action probability weighting
actor_loss = (action_probs * ((alpha * log_pi) - min_qf_values)).mean()
```

<Note>
  `sac_atari.py` uses `--target-entropy-scale=0.89` (vs. `0.98` in the original paper) for improved stability during longer training runs. Updates are performed every nth environment step rather than every step.
</Note>
