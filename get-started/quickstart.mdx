CleanRL algorithms are plain Python scripts you run directly from the command line. There is no framework to initialise and no configuration file to learn — every hyperparameter is a CLI flag. This page shows you how to launch your first training run, inspect the output, and move on to other environments and algorithms.

## Run PPO on CartPole-v1

After [installing CleanRL](/get-started/installation), you can start a training run immediately. CleanRL provides two workflows depending on how you manage your virtual environment.

<Tabs>
  <Tab title="uv run">
    `uv run` launches the script inside the project's managed environment without manually activating it first.

    ```bash
    uv run python cleanrl/ppo.py \
        --seed 1 \
        --env-id CartPole-v1 \
        --total-timesteps 500000
    ```
  </Tab>
  <Tab title="uv venv">
    Activate the virtual environment first, then run the script as normal Python. Each step must be run separately.

    ```bash
    uv venv
    ```

    ```bash
    python cleanrl/ppo.py \
        --seed 1 \
        --env-id CartPole-v1 \
        --total-timesteps 500000
    ```

    <Tip>
      We recommend the `uv venv` workflow for development. Once activated, your shell prompt shows a prefix like `(cleanrl-iXg02GqF-py3.9)`, and you can run `tensorboard` and other tools in the same shell.
    </Tip>
  </Tab>
</Tabs>

## Key CLI flags

Every CleanRL script exposes its full configuration as CLI flags. The flags below are common across most algorithm scripts.

| Flag | Default | Description |
| --- | --- | --- |
| `--env-id` | `CartPole-v1` | The Gymnasium environment ID |
| `--seed` | `1` | Random seed for reproducibility |
| `--total-timesteps` | `500000` | Total environment steps to run |
| `--learning-rate` | `2.5e-4` | Optimizer learning rate |
| `--num-envs` | `4` | Number of parallel environments |
| `--capture_video` | `False` | Save gameplay videos to `videos/` |
| `--track` | `False` | Enable Weights & Biases tracking |
| `--cuda` | `True` | Use GPU if available |

## Visualize training metrics with Tensorboard

CleanRL writes all training metrics to the `runs/` directory using Tensorboard's summary format by default — no extra configuration required. Open a second terminal and run:

```bash
tensorboard --logdir runs
```

Then open `http://localhost:6006` in your browser to see episodic return, learning rate, loss curves, and more updating in real time.

<Note>
  If you are using the `uv venv` workflow, run `tensorboard` inside the activated shell so the command resolves to the correct environment.
</Note>

## Capture gameplay videos

Add `--capture_video` to any run to save MP4 recordings of the agent's gameplay to `videos/{run_name}/`.

```bash
python cleanrl/ppo.py \
    --seed 1 \
    --env-id CartPole-v1 \
    --total-timesteps 500000 \
    --capture_video
```

## Run other algorithms and environments

CleanRL ships algorithm variants as separate single-file scripts. Activate your environment (`uv venv`) and run any of the following:

<CodeGroup>

```bash Classic control
# DQN on CartPole
python cleanrl/dqn.py --env-id CartPole-v1

# PPO on CartPole
python cleanrl/ppo.py --env-id CartPole-v1

# C51 on CartPole
python cleanrl/c51.py --env-id CartPole-v1
```

```bash Atari (requires .[atari])
uv pip install ".[atari]"

python cleanrl/dqn_atari.py --env-id BreakoutNoFrameskip-v4
python cleanrl/ppo_atari.py --env-id BreakoutNoFrameskip-v4
python cleanrl/sac_atari.py --env-id BreakoutNoFrameskip-v4
```

```bash EnvPool Atari (Linux only, requires .[envpool])
uv pip install ".[envpool]"

# 3-4x speed-up with side-effects-free vectorisation
python cleanrl/ppo_atari_envpool.py --env-id BreakoutNoFrameskip-v4
```

```bash Procgen (requires .[procgen])
uv pip install ".[procgen]"

python cleanrl/ppo_procgen.py --env-id starpilot
python cleanrl/ppg_procgen.py --env-id starpilot
```

</CodeGroup>

<Tip>
  Each script is fully self-contained. You can read `cleanrl/ppo.py` from top to bottom to understand every implementation detail of PPO — no need to trace through a modular library.
</Tip>
