Random Network Distillation (RND) is an exploration bonus that can be layered on top of any RL algorithm. A randomly initialized target network and a trained predictor network are maintained. The predictor's error on new observations serves as an intrinsic curiosity reward: high error means the observation is novel, so the agent is rewarded for exploring it. As the predictor learns to match the target, intrinsic rewards decay for visited states.

This implementation pairs RND with [PPO](/rl-algorithms/ppo) (following the original paper) and uses [EnvPool](https://github.com/sail-sg/envpool) for highly efficient vectorized Atari environments. It demonstrates strong results on hard-exploration games such as Montezuma's Revenge.

**Original paper:**

- [Exploration by Random Network Distillation](https://arxiv.org/abs/1810.12894) — Burda et al., 2019

## `ppo_rnd_envpool.py`

This implementation uses EnvPool's blazing-fast Atari vectorized environment with standard convolutional preprocessing. It works with Atari's pixel `Box` observation space (210×160×3) and `Discrete` action spaces.

<Warning>
  `ppo_rnd_envpool.py` only works on Linux. EnvPool does not provide wheels for Windows or macOS. See [envpool's available wheels](https://pypi.org/project/envpool/#files).
</Warning>

<Warning>
  EnvPool (≤0.6.4) has a known behavior difference from gym's vectorized environments: on episode termination, `env.step()` returns the terminated state `s_last` rather than the initial state of the new episode `s_new`. This causes the observation to be off by one step. It does not appear to affect performance in practice. See [sail-sg/envpool#194](https://github.com/sail-sg/envpool/issues/194) for updates.
</Warning>

### Key implementation details

1. **Observation normalization initialization** — Before training begins, a random agent steps through the environment for `num_steps * num_iterations_obs_norm_init` steps to warm up the running mean and variance used to normalize observations for the RND networks. This follows the [original implementation](https://github.com/openai/random-network-distillation/blob/f75c0f1efa473d5109d487062fd8ed49ddce6634/run_atari.py#L69).

2. **Sticky actions** — EnvPool is configured with a repeat-action probability to replicate the sticky action setting used in the original paper. This makes the exploration problem harder and more realistic.

3. **Separate discount rates** — The intrinsic and extrinsic rewards use different discount factors (`int_gamma=0.99` vs. `gamma=0.999`), since intrinsic rewards do not need to be propagated as far into the future.

4. **Custom episode statistics** — A custom `RecordEpisodeStatistics` wrapper is used to track both extrinsic and intrinsic episodic returns correctly under EnvPool's episode handling behavior.

### Logged metrics

In addition to the standard PPO metrics, `ppo_rnd_envpool.py` logs:

- `charts/episode_curiosity_reward` — Intrinsic (curiosity) reward accumulated per episode.
- `losses/fwd_loss` — Mean squared error between the predictor and target networks. This is also a proxy for the intrinsic reward magnitude in the current batch.
