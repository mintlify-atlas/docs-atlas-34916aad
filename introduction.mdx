CleanRL is a deep reinforcement learning library built for researchers and practitioners who want to understand exactly how algorithms work. Unlike modular RL libraries, every implementation detail of an algorithm variant lives in a single, self-contained Python file — making it easy to read, modify, and extend without navigating complex class hierarchies.

## Why CleanRL?

CleanRL is designed around a single principle: **every implementation detail should be visible in one file**. This makes it ideal for:

- **Learning**: Read `ppo_atari.py` (340 lines) to understand exactly how PPO works on Atari games — no hunting through multiple modules
- **Research**: Prototype new ideas by modifying a single file, with minimal boilerplate to work around
- **Reproducibility**: Seeded experiments, Tensorboard logging, and W&B integration are built in from the start

<Note>
CleanRL is **not** a modular library and is not meant to be imported as a dependency. It is a collection of reference implementations. If you need a production-grade modular library, consider [Stable-Baselines3](https://github.com/DLR-RM/stable-baselines3).
</Note>

## Get started

<Steps>
  <Step title="Install CleanRL">
    Clone the repository and install dependencies with `uv`:
    ```bash
    git clone https://github.com/vwxyzjn/cleanrl.git && cd cleanrl
    uv pip install .
    ```
  </Step>
  <Step title="Run your first experiment">
    Train a PPO agent on CartPole in under a minute:
    ```bash
    uv run python cleanrl/ppo.py \
        --seed 1 \
        --env-id CartPole-v1 \
        --total-timesteps 50000
    ```
  </Step>
  <Step title="Visualize results">
    Open Tensorboard to view training metrics:
    ```bash
    tensorboard --logdir runs
    ```
  </Step>
  <Step title="Track with Weights & Biases">
    Add the `--track` flag to log your run to W&B:
    ```bash
    uv run python cleanrl/ppo.py \
        --env-id CartPole-v1 \
        --total-timesteps 50000 \
        --track \
        --wandb-project-name my-cleanrl-project
    ```
  </Step>
</Steps>
