description: "Off-policy deterministic actor-critic with replay buffer and target networks for continuous control. Covers PyTorch and JAX variants for MuJoCo."

Deep Deterministic Policy Gradient (DDPG) extends DQN to continuous action spaces by introducing a deterministic actor that directly outputs actions. The algorithm combines a replay buffer and target networks (borrowed from DQN) with an actor-critic architecture to learn stable policies in continuous control tasks.

Compared to [TD3](/rl-algorithms/td3), DDPG uses only a single critic network and does not apply target policy smoothing noise or delayed actor updates. In practice, TD3 tends to outperform DDPG on most benchmarks due to these additions.

**Original paper:**

- [Continuous control with deep reinforcement learning](https://arxiv.org/abs/1509.02971) — Lillicrap et al., 2016

## `ddpg_continuous_action.py`

This variant is based on [sfujim/TD3's OurDDPG.py](https://github.com/sfujim/TD3/blob/master/OurDDPG.py) and targets continuous control environments with `Box` observation and action spaces.

### Differences from the original paper

| Detail | CleanRL's `ddpg_continuous_action.py` | Lillicrap et al. (2016) |
|--------|--------------------------------------|-------------------------|
| Exploration noise | Gaussian N(0, 0.1) | Ornstein-Uhlenbeck (θ=0.15, σ=0.2) |
| Network width | 256 units per hidden layer | 400/300 units per hidden layer |
| Learning rate | 3e-4 for both actor and critic | 1e-4 (critic), 1e-3 (actor) |
| Batch size | 256 | 64 |
| Tau | 0.005 | 0.001 |

### Key implementation details

CleanRL's DDPG handles asymmetric action space bounds (e.g., `Humanoid-v2` with `[-0.4, 0.4]`) using `action_scale` and `action_bias`:

```python
class Actor(nn.Module):
    def __init__(self, env):
        super().__init__()
        self.fc1 = nn.Linear(np.array(env.single_observation_space.shape).prod(), 256)
        self.fc2 = nn.Linear(256, 256)
        self.fc_mu = nn.Linear(256, np.prod(env.single_action_space.shape))
        # action rescaling
        self.register_buffer(
            "action_scale", torch.tensor((env.action_space.high - env.action_space.low) / 2.0, dtype=torch.float32)
        )
        self.register_buffer(
            "action_bias", torch.tensor((env.action_space.high + env.action_space.low) / 2.0, dtype=torch.float32)
        )

    def forward(self, x):
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        x = torch.tanh(self.fc_mu(x))
        return x * self.action_scale + self.action_bias
```

<Note>
  The reference implementation used MuJoCo v1 environments and evaluated in deterministic mode (no exploration noise). CleanRL uses MuJoCo v4 and reports training-time episodic return.

## `ddpg_continuous_action_jax.py`

This variant uses [JAX](https://github.com/google/jax), [Flax](https://github.com/google/flax), and [Optax](https://github.com/deepmind/optax) instead of PyTorch. It achieves roughly 2.5–4× faster wall-clock training than the PyTorch variant.

### Installation and usage

<Tabs>

    ```bash

    uv run python cleanrl/ddpg_continuous_action_jax.py --help
    uv run python cleanrl/ddpg_continuous_action_jax.py --env-id Hopper-v4
    uv run python cleanrl/ddpg_continuous_action_jax.py --env-id HalfCheetah-v4
    ```

  </Tab>
  <Tab title="pip">

  </Tab>
</Tabs>
